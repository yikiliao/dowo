
軟體開發工具的選擇:

Ch4、軟體開發工具的選擇

Appserv是將apache、PHP、MySQL、PHPMySQL包在一起的軟體，有了AppServ之後，我們便可輕易的在Windows作業系統中安裝好整套的Apache、PHP、MySQL等網頁伺服器套件，只要執行完AppServ安裝程序，便可將所需的伺服器功能一次全部稿定。

4-1 PHP應用

PHP是自由軟體，特別適用於網路系統發展，可以加入HTML 用途廣泛的scripting 語言。要知道PHP普及狀況可以看Netcraft 調查報告。

PHP: 20,917,850 domains, 1,224,183 IP addresses

Source: Netcraft

PHP同時也是Apache網路伺服器最普遍的使用者。

(資料來源：http://www.php.net/usage.php)

由整個走勢圖看來PHP經3年盤整有浴火重生，再度快速成長趨勢，現在學習PHP使用正是時候。

4-2 MySQL資料庫應用

MySQL也是免費的資料庫軟體，是由MySQL AB公司所開發的資料庫伺服器，可在許多平台上運作，如：Linux、Windows、Sun Solaris ... 等，而且也可以連結C、C++、Java、Perl、PHP等語言，且支援微軟的ODBC規格的資料庫整合。

對不同使用者設定權限，在資料庫中必須依權限的設定才能使用及進入資料表，提高了安全性，權限的使用也是MySQL特別的地方。

MySQL可跨越不同的作業平台，在Windows系統下作業，那並不影響資料庫的完整性，設定也幾乎和在Linux系統下是一樣的，若你使用Linux系統，也可以在Windows系統下編輯程式，再傳輸到Linux系統，以截長補短，運用Windows發展工具多，Linux系統穩定而速度快的特色。

4-3 Appserv運行環境建立，自己安裝

Appserv是將apache、PHP、MySQL、PHPMySQL包在一起的軟體

對一般人而言，要架一個網頁伺服器似乎是一個遙不可及的夢想，但是AppServ把這件事變得很簡單。AppServ的安裝過程非常的容易，就如同一般的應用程式一樣，只要順著「下一步」，一步一步進行，就能裝好。裝好AppServ就可以將自己製作的網頁擺放上去，或是用phpBB、WordPress架設自己的論壇或是Blog了

AppServ 是含Apache/PHP/MySQL 的自由開放的套裝軟體。

它可以讓您

＊容易的建立網路伺服器和資料庫伺服器。

＊為那些開始使用client/server型態的程式撰寫。

＊為web base 程式發展使用PHP & MySQL 。

＊ 為容易地將被端起對其它平臺譬如WindowZ 的編程技術

- ＊ 簡單照步驟執行，而且步驟少，不需多少步驟即可完成，，不需花費設施和配置。

＊在您完成了安裝之後，即可執行。

4-3-1 到那裡下載？

如果要在網路上下載，建議到官方網站，可以下載到最新版本；網路上其他的站台，不一定會有最新版本。不過在這?也要給各位一個建議，軟體不見得一定要追求最新版。很多軟體可能每個星期都會出更新版，那是不是又要移除舊版，換上更新版呢？只要軟體穩定，沒有重大的漏洞或是重大改版，其實是可以不用更新版本的。尤其「管理資訊系統」類應用軟體對系統穩定度的要求相對於網站而言較高，所以穩定成熟版本，更甚於最新版本。

4-3-3 程式擺到那裡？

AppServ其實很方便，web部分只要把程式、圖片放在C:\AppServ\www路徑下， 所有的網頁都可以跑了；不過為了方便管理維護，最好在C:\AppServ\www底下再加個資料夾，若是自己架server，上傳就是將檔案放在:\AppServ\www底下或加上其底下的資料夾就可以了。

4-3-4 會有什麼問題？如何解決？

1.裝不上？

＊解決方法：

1、先關閉有的沒有的軟體，例如防火牆、防毒軟體、防間諜軟體再安裝AppServ，安裝好AppServ再逐一開放，其它軟體，或更換其它軟體。

2.無法顯示網頁或空白頁

＊解決方法：

1、請先確認是否有裝IIS，按【控制台】→【系統管理工具】→【服務】找看看有無IIS這一項，並將它停掉。

1.移除AppServ內容。

2.重開機後，將C：\APPSERV資料夾刪除。

3.從頭重新安裝再試試看吧。

2. 裝上了，無法運作，怎麼辦？

＊解決方法：

1.從【開始】→【控制台】→【系統管理工具】→【事件檢視器】→【應用程式】或【系統】；在【錯誤類型】列上按滑鼠右鍵，查閱【內容】，瞭解內容描述，再針對問題解決。

2. 進入C:\AppServ\apache2/logs 底下檢查【access】及【error】兩項文件，瞭解運作情形，再針對問題解決。

3.從【開始】→【所有程式】→【AppServ】→【Control Server by Service】→

【Apache Monitor】檢查Apache有沒有安裝成功，看Apache 啟動狀況是否亮綠燈；如果亮紅燈則不正常，可以在【Apache Service Monitor 】畫面下，按【restart】進入【服務（本機）】，在【apache2】點選【內容（R）】→再按【修復】。如果還不是綠燈，則有可能是Apache與電腦環境相衝，建議改用其他版本試試。其圖示如下：
